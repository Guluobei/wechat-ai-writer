# 模块 3：大纲构建（Outliner）

> 结构 + 节奏 + 配图标注。基于调研素材设计文章结构。★ 确认点 1

## 模块职责

基于调研报告设计文章结构，生成多级大纲，进行逻辑与节奏校验，标注配图与差异化角度，设计开头钩子与结尾收束。输出后暂停，等待用户确认（确认点 1）。

## 输入

| 输入项 | 说明 |
|--------|------|
| 调研报告 | `ArticleProject.research`，含事实、数据、观点、案例、竞品覆盖分析 |

> 前置校验：若无调研报告，提示「大纲需要调研素材。先运行调研，或直接粘贴你的素材给我」。

## 输出

多级大纲，写入 `ArticleProject.outline` 字段。

大纲包含：文章类型、一级标题与二级要点、每节篇幅占比与阅读重点、配图标注、开头钩子、结尾收束、差异化标注。

## 执行步骤

### 1. 匹配结构模板

根据文章类型匹配对应结构模板，作为大纲骨架：

| 文章类型 | 说明 |
|----------|------|
| `tech_analysis` | 技术解读：背景 → 原理拆解 → 关键创新 → 意义与局限 |
| `product_review` | 产品分析：产品概览 → 核心功能体验 → 定位与竞品 → 评价与建议 |
| `science_explainer` | 科普：现象/问题 → 概念解释 → 原理通俗化 → 影响与展望 |
| `industry_insight` | 行业影响：事件/趋势 → 驱动因素 → 格局影响 → 趋势判断 |

### 2. 生成多级大纲

- 生成一级标题 + 二级要点结构。
- 将调研报告中的事实、数据、案例分配到对应章节，确保素材有用武之地。

### 3. 逻辑校验

逐项检查并修正：

- **论点支撑**：每个论点是否有对应素材支撑，无支撑的论点补充素材或删除。
- **维度完整**：关键维度是否遗漏（如技术解读是否漏了局限性、产品分析是否漏了竞品对比）。
- **过渡自然**：章节之间是否有逻辑断层，必要时增加过渡段落。

### 4. 节奏设计

为每节标注：

- **预计篇幅占比**：如「20%」，全篇占比合计 100%。
- **阅读重点**：本节最想让读者记住的一句话或一个结论。

### 5. 配图标注

为每节标注配图类型与配图描述：

| 配图类型 | 说明 |
|----------|------|
| `ai_generated` | AI 生成配图，附描述供模块 5 生成 |
| `screenshot` | 产品/界面截图，标注所需画面与取图指引 |
| `data_chart` | 数据图表，标注数据来源与图表类型 |
| `none` | 该节不需要配图 |

### 6. 钩子设计

- **开头钩子**：用提问、悬念、反常识或场景化开场抓住注意力。
- **结尾收束**：呼应开头，引导互动（评论话题）或关注（关注理由）。

### 7. 差异化标注

基于调研报告中的竞品覆盖分析，标注：

- **本文独有角度**：竞品未覆盖、本文重点展开的部分。
- **本文独有信息**：竞品未提及的数据、案例、观点。

## 门控：确认点 1

大纲输出后**必须暂停**，向用户展示并等待确认：

```
📋 大纲已完成，请确认：
  [展示大纲结构、各节要点、配图标注、差异化角度]
  → 回复「确认」继续撰写
  → 回复「修改：XX」调整大纲
  → 回复「重做」重新生成大纲
```

- 用户「确认」：进入模块 4 撰写。
- 用户「修改：XX」：按反馈调整后重新展示。
- 用户「重做」：保留调研报告，重新生成大纲。

## 注意事项

1. **素材驱动结构**：大纲的每个论点都要能追溯到调研报告中的素材，不要空写。
2. **篇幅占比合理**：开头钩子和结尾收束占比不宜过大，核心内容应占主体。
3. **配图服务于内容**：配图标注须说明该图表达什么，不能为配图而配图。
4. **差异化要落地**：差异化标注必须基于竞品扫描结果，不能泛泛而谈「本文更有深度」。
5. **结构不要过深**：一般控制在一级标题 + 二级要点，层级过深影响公众号阅读体验。
6. **门控不可跳过**：确认点 1 是流程硬性节点，未获用户确认不得进入撰写环节。
7. **回退保留调研**：用户要求重做时只重做大纲，调研报告保留不动。
