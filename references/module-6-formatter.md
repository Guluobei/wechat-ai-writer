# 模块 6：排版美化（Formatter）

> 初稿 + 配图 → 公众号编辑器可直接粘贴的 HTML。全部内联样式，移动端优先适配。

## 模块职责

- 将 Markdown 初稿转换为公众号编辑器可直接粘贴的 HTML
- 应用排版模板样式，全部内联样式（公众号编辑器不支持外部 CSS）
- 按配图规划插入图片，处理图文混排与图片说明
- 将引用标记转为公众号友好的引用呈现方式
- 移动端适配优化
- 预览自检，确保格式可用

## 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 初稿正文 | `ArticleProject.draft` | Markdown 格式正文 |
| 配图清单 | `ArticleProject.images` | 封面图 cover + 文中插图 inline[]，含路径、caption、status |
| 排版模板 | `assets/default-template.html` | 默认公众号 HTML 模板，提供基础样式 |
| 项目级模板（可选） | `/workspace/projects/<article_id>/config/template.html` | 项目级覆盖模板 |
| 用户级模板（可选） | `/workspace/config/wechat-ai-writer/template.html` | 用户自定义模板 |
| 引用呈现偏好 | 用户选择 / 默认 | 脚注式 / 文末参考式 |

模板加载顺序：项目级 → 用户级 → 内置默认，缺失时用默认模板并标注「使用默认配置」。

## 输出

- 成品 HTML 写入 `ArticleProject.formatted` 字段
- 同时保存到 `/workspace/projects/<article_id>/final.html`
- 成品 HTML 要求：全部内联样式，可直接粘贴到公众号编辑器

## 执行步骤

### 步骤 1：Markdown → 公众号 HTML

将初稿 Markdown 转换为 HTML，核心要求是**全部内联样式**（公众号编辑器会过滤 `<style>` 标签和外部 CSS）。

- 标题（h1-h4）、段落、列表、引用、加粗、行内代码、代码块逐一转换
- 每个元素的内联样式直接写在 `style` 属性上
- 保留语义结构，便于模板样式覆盖

### 步骤 2：应用排版模板

使用 `assets/default-template.html`（或更高优先级模板）的样式：

- 读取模板中定义的各元素样式规则（标题字号/颜色、正文行高、引用块样式、代码块样式等）
- 将样式规则以内联方式应用到每个 HTML 元素
- 模板缺失时使用内置默认值（见步骤 5 移动端规格）并标注「使用默认配置」

### 步骤 3：图片嵌入与上传清单

配图按大纲规划插入正确位置，处理图文混排和图片说明。

**关键：公众号不支持本地文件路径**

公众号编辑器要求图片手动上传，不支持本地路径（如 `/workspace/projects/xxx/images/cover.png`）。因此排版时：

1. HTML 中图片位置用占位符标记：`<img src="【请上传：cover.png】" alt="封面图" .../>`
2. 同时生成**图片上传清单**，列出每张图的文件路径、对应位置、说明文字
3. 在确认点3展示时，告知用户需要手动上传的图片数量和清单

图片上传清单格式：
```
图片上传清单（共 N 张）：
1. 封面图 — 文件：/workspace/projects/xxx/images/cover.png — 粘贴到公众号后点击上传此图
2. 图1（第2节）— 文件：/workspace/projects/xxx/images/fig1.png — 粘贴到公众号后点击上传此图
...
```

### 步骤 4：引用呈现

将初稿中的内联引用标记转为公众号友好样式，提供两种方式（用户可选，默认脚注式）：

- **脚注式**：正文中引用处用上标编号 `[1]`，段末或文末附简短出处
- **文末参考式**：正文中引用处用编号标记，文末统一列参考文献列表

引用信息来自 `ArticleProject.research.citations`，确保每条引用可追溯到来源。

### 步骤 5：移动端适配

公众号阅读以移动端为主，按以下规格优化：

| 元素 | 规格 |
|------|------|
| 正文字号 | 16px |
| 行高 | 1.8 |
| 段落间距 | 适当留白，段间 1em 左右 |
| 标题字号 | h1 22px / h2 20px / h3 18px / h4 16px |
| 图片宽度 | 适配移动端宽度，最大 100%，居中 |
| 代码块 | 横向滚动或自动换行，字号 14px |
| 引用块 | 左侧色条 + 浅底，字号 15px |

### 步骤 6：预览自检

生成成品 HTML 后自检三项：

1. **格式错乱检查**：内联样式是否生效、是否有未闭合标签、列表/引用嵌套是否正确
2. **图片缺失检查**：清单中 `success` 状态的图片是否都已嵌入，占位/跳过的图片是否有明确标注
3. **引用完整性检查**：正文引用标记是否都有对应出处，文末参考列表是否完整

自检通过后将成品写入 `ArticleProject.formatted` 并保存到 `final.html`。

## 降级方案

| 失败点 | 自动降级 | 用户感知（三段式） |
|--------|---------|------------------|
| 配图缺失 | 留空或文字占位 | 「第 2 张配图缺失，已留占位，发布前可补图」 |
| 模板缺失 | 用内置默认样式 | 「未找到排版模板，已用默认样式排版」 |
| 引用来源缺失 | 保留标记但标注「待核实」 | 「第 3 条引用来源缺失，已标注待核实」 |
| Markdown 解析异常 | 降级为纯文本段落 | 「部分格式解析异常，已按纯文本处理，建议检查」 |
| 图片本地路径不可用于公众号 | HTML 中用占位符标记位置 | 「图片本地路径不可用于公众号，所有图片均需用户手动上传，HTML 中已用占位符标记位置」 |

## 注意事项

1. **全部内联样式**：公众号编辑器会过滤 `<style>` 标签和 class，所有样式必须写在 `style` 属性上，这是排版可用的硬性前提
2. **移动端优先**：阅读场景以手机为主，字号、行高、图片宽度都按移动端优化，不要按桌面端设计
3. **图片占位要可见**：降级占位必须显眼可识别（如「[待补图：产品主界面截图]」），便于发布前清单环节发现
4. **引用不丢来源**：引用呈现要保证可追溯，文末参考式需完整列出来源，脚注式需对应出处
5. **成品可直接粘贴**：`final.html` 的内容应能直接复制粘贴进公众号编辑器且样式不丢，这是验收标准
6. **不擅自删改正文**：排版只做格式转换与图片嵌入，不修改正文文字内容；发现文字问题应留给校对模块处理
7. **预览自检必做**：自检是进入校对模块前的最后关口，格式错乱会显著影响读者体验，不可跳过
