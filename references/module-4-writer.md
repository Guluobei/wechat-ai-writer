# 模块 4：正文撰写（Writer）

> 调性适配 + 风格档案 + 实时事实核查 + 四层自检。按大纲和素材生成初稿。★ 确认点 2

## 模块职责

依据大纲与调研素材，参照风格档案生成完整初稿。写作过程中实时事实核查，处理专业术语，控制过渡与节奏。初稿完成后执行四层自检体系。输出后暂停，等待用户确认（确认点 2）。

## 输入

| 输入项 | 说明 |
|--------|------|
| 大纲 | `ArticleProject.outline`，含结构、节奏、配图标注、钩子与收束 |
| 调研报告 | `ArticleProject.research`，含事实、数据、观点、案例、引用 |
| 风格档案 | `assets/default-style-profile.md`（v2.0），AI 行业调性基线 |

> 前置校验：若无大纲，提示「撰写需要大纲。先运行大纲构建，或直接告诉我文章结构」。

## 输出

完整初稿，写入 `ArticleProject.draft` 字段，结构如下：

```json
{
  "content_markdown": "完整初稿 Markdown 文本",
  "word_count": 3500,
  "article_archetype": "调查实验型 | 产品体验型 | 现象解读型 | 工具分享型 | 方法论分享型 | 横评对比型",
  "inline_citations": [
    {"ref": "来源名称", "url": "链接", "position": "段落位置描述"}
  ],
  "self_check": {
    "L1_hard_rules": "passed | failed",
    "L2_style": "passed | failed",
    "L3_content": "passed | failed",
    "L4_human_feel": "passed | failed",
    "issues_found": ["问题描述列表"],
    "fixes_applied": ["已修复问题列表"]
  }
}
```

- `content_markdown`：完整初稿正文，Markdown 格式，含内联引用标记（如 [1]、[2]）
- `word_count`：正文字数统计
- `article_archetype`：文章原型，撰写前判断（见步骤 0）
- `inline_citations`：内联引用列表，每条含来源名称、链接和位置描述
- `self_check`：四层自检结果（见步骤 7）

## 执行步骤

### 步骤 0：判断文章原型

写之前先判断本文属于哪种原型，决定写法重心：

| 原型 | 核心 | 写法重心 |
|------|------|---------|
| 调查实验型 | 亲自下场做一件事，边做边报道发现 | 过程叙事 + 层层发现 + 每一步带真实反应 |
| 产品体验型 | 拿到产品实际使用，带着读者一起体验 | 场景驱动 + 真实感受 + 与其他产品自然对比 |
| 现象解读型 | 观察到一个现象，层层深入分析 | 观察→好奇→研究→哲学升维 |
| 工具分享型 | 分享实用工具/Prompt，用个人故事包裹 | 个人故事铺垫→工具展示→效果惊艳 |
| 方法论分享型 | 系统性分享经验和方法论 | 每节落到可执行行动 + 坦诚学习成本 + 谦逊铺垫 |
| 横评对比型 | 多产品/方案系统性对比 | 逐一展示（升番逻辑）+ 对比表 + 选型决策 |

原型判断结果写入 `article_archetype` 字段，后续步骤均参照对应原型的写法重心。

### 步骤 1：调性适配

根据文章类型切换写作风格：

| 文章类型 | 调性倾向 |
|----------|----------|
| `tech_analysis` | 专业为底，用聊天感降低门槛 |
| `product_review` | 务实导向，强调"跟我一起玩"的体验感 |
| `science_explainer` | 通俗生动，善用类比与生活化语言 |
| `industry_insight` | 有判断力，逻辑严密但不写成研报 |

### 步骤 2：风格档案参照

始终参照 `assets/default-style-profile.md`（v2.0），保证跨文章调性一致。v2.0 风格档案包含：
- **风格内核**（16 种技法）：节奏感、论述中的故意打破、知识"随手掏"、私人视角、判断力、对立面理解、情绪表达、亲自下场感、人物画像法、文化升维、句式断裂、回环呼应、谦逊铺垫法、读者直呼法、疑问句节奏作用、层层剥开修辞、英雄之旅叙事弧、反向论证、逐一展示法
- **合规底线**：不夸大、不编造、术语加注、事实与推测分离
- **写作禁忌**：套话禁区、高频踩雷词、假设性例子、空泛工具名、教科书开头

若存在用户级风格档案（`/workspace/config/wechat-ai-writer/style-profile.md`），优先采用并覆盖默认项。

### 步骤 3：素材织入

- 将事实、数据、案例自然织入正文，避免堆砌罗列。
- 引用以公众号友好方式呈现：行文内自然提及来源，关键数据可在句末简注出处，不使用学术式脚注。
- 观点性内容须标注来源或持有者，不把他人观点写成自家结论。
- 知识点以"聊着聊着顺手掏出来"的方式呈现，不用"下面我来介绍"。

### 步骤 4：专业术语处理

- AI 术语首次出现时加注简短解释（括号或同位语）。
- 重复出现不再重复加注。
- 面向泛读者的科普类文章，术语解释应更通俗。

### 步骤 5：实时事实核查

- 写作过程中实时核对引用与调研报告是否一致。
- 引用无出处或与调研报告冲突的，标记「待核实」。
- 标记位置清晰可辨，便于后续校对模块（模块 7）复核。

### 步骤 6：过渡与节奏

> **方法论补充**：本节融入了内容写作角度方法论（参见 `references/content-angle-methodology.md`）的节奏设计技法。好的节奏像呼吸，有松有紧——一篇好内容一定有波峰有波谷，纯平的内容像说明书，读者读完没有任何情绪起伏。

**6.0 写前情绪地图绘制**

正式撰写前，先为文章画一条情绪曲线。标出每个段落/章节想让读者处于什么情绪状态（如：好奇→震惊→共鸣→愤怒→释然→升华），确保有起伏而非一条直线。如果发现连续三个段落情绪处于同一水平，需要调整节奏。

**6.1 基础节奏控制**

- 控制段落长度，避免大段文字墙。很多段落可以只有 1-2 句话。
- 章节间过渡自然，符合大纲节奏设计（篇幅占比）。
- 信息密度适中，重点处适当展开，铺垫处适度精简。
- **扣主线句**高频出现：每次围绕主线偏出去一点点，再用一句话拉回来。
- 小标题使用根据原型灵活处理：横评对比型/方法论分享型可用小标题，叙事型尽量不用，靠口语化转场句衔接。

**6.2 升番技巧（Sketch Escalation）**

当文章中需要排布多个案例/论据时，采用升番逻辑——每个案例要比前一个更有冲击力。

- 第一番：让读者注意到（引入第一个案例，建立基线）
- 第二番：让读者更惊讶（第二个案例冲击力升级）
- 第三番：给读者完全没想到的东西（最有冲击力的案例放最后）

> 来自《一年一度喜剧大赛》的 sketch 喜剧技巧：三个爆点不断往上翻，给观众的情绪一浪比一浪高。应用到文章中就是案例的排列顺序——最弱的放前面，最炸的留到最后。

**6.3 弱者思维强化**

在涉及判断和观点表达的段落，保持谦逊姿态。承认自己的不确定性和局限性，用"我也在摸索""可能有些想法还不成熟"来卸掉说教感。

> 弱者思维不是虚伪的谦虚，是真实的不确定感——承认自己在广义社会下并不是最牛的，只是爱分享了一点，反而更真诚、更可信。

### 步骤 7：四层自检

初稿完成后，必须跑完整个四层质检流程。自检结果写入 `self_check` 字段。

**L1 硬性规则检查（自动扫描层）：**
- L1-1 禁用词扫描：搜索"说白了""意味着什么""本质上""换句话说""不可否认""综上所述"等，出现则替换
- L1-2 禁用标点扫描：中英文标点混用、全半角混用
- L1-3 结构性套话扫描："让我们来看看""在当今…的时代""随着…的发展"
- L1-4 工具名检查：所有 AI 工具/产品使用具体名称，无"AI 工具""某个模型"
- L1-5 合规扫描：夸大/绝对化表述、未标注来源数字、未区分事实与推测
- 通过标准：五项零命中

**L2 风格一致性检查（模式匹配层）：**
- L2-1 开头检查：从具体事件切入？第一句让读者想继续？无教科书开头？
- L2-2 节奏与结构：长短句交替？一句话独立成段（至少 2-3 次）？偏离主线后有扣主线句？疑问句制造节奏？
- L2-3 口语化检查：使用了 5-8 个口语化表达？有论述中的故意打破？有自嘲或承认不足？
- 通过标准：L2-1 全部通过，L2-2 至少 3/4，L2-3 至少 2/3

**L3 内容质量检查（深度审查层）：**
- L3-1 观点支撑：每个核心观点有具体的人/场景/细节/数据支撑？
- L3-2 知识输出：知识点以"随手掏"方式呈现？无"下面我来介绍"？
- L3-3 文化升维：至少一处从具体事件连接到更大的参照物？
- L3-4 对立面与同理心：讲观点时先理解对方立场？
- L3-5 文章类型专项：根据原型做针对性检查（见风格档案 L3-5）
- 通过标准：L3-1 和 L3-2 必须通过，L3-3 至 L3-5 至少通过相关项

**L4 活人感终审（最终人格层）：**
- 核心问题："读完这篇文章，感觉是一个懂 AI 的人在认真跟我聊一件他觉得重要的事，还是 AI 在输出信息？"
- L4-1 温度感：体感记忆 vs 知识性描述？
- L4-2 独特性：只有这个作者才会写出的角度？
- L4-3 姿态：懂行的人在聊事 vs 导师在教学生/品牌在做营销？
- L4-4 心流：从头到尾读，有没有注意力断掉的地方？
- 通过标准：整体感觉"像真人写的"。任何一项"AI 味太重"就返工。

**自检后处理：**
- L1 发现的问题直接修复，记入 `fixes_applied`
- L2/L3 发现的问题尽量修复；无法修复的记入 `issues_found`，在交付时告知用户
- L4 发现的问题需要返工重写相关段落，不能跳过

## 降级方案

- **事实核查无引用支撑**：标记「待核实」继续写作，不中断流程，并在初稿交付时告知用户「第 N 段数据未找到出处，已标注待核实」。
- **L4 活人感不通过**：重写 AI 味重的段落，用更口语、更私人、更不完美的方式。如果时间紧迫，至少修复最严重的 1-2 处，其余记入 `issues_found`。

## 门控：确认点 2

初稿输出后**必须暂停**，向用户展示并等待确认：

```
初稿已完成（约 XXXX 字），文章原型：XX型，四层自检结果：L1✅ L2✅ L3✅ L4✅
  [展示初稿全文或摘要]
  [如有未修复问题，列出 issues_found]
  → 回复「确认」继续配图
  → 回复「修改：XX」调整初稿
  → 回复「重做」重新撰写
```

- 用户「确认」：进入模块 5 配图。
- 用户「修改：XX」：按反馈调整指定部分后重新展示。
- 用户「重做」：保留大纲与调研，重新撰写。

## 注意事项

1. **风格档案优先级**：用户级风格档案 > 默认风格档案 v2.0，二者冲突时以用户级为准。
2. **不脱离大纲**：正文须覆盖大纲所有章节，新增内容不得偏离大纲结构；如确需调整结构，应提示用户回到模块 3。
3. **不编造素材**：调研报告未提供的事实、数据、案例不得自行杜撰，缺素材处标记「待核实」或留出占位。
4. **引用必须可溯**：正文中出现的所有数据与关键事实，须能在调研报告引用列表中找到出处。
5. **术语加注仅首次**：避免重复加注造成阅读打断。
6. **「待核实」要显眼**：标记格式统一且醒目，便于校对环节集中处理。
7. **四层自检不可跳过**：L1-L4 是初稿交付的前置条件，未完成自检不得提交给用户确认。
8. **门控不可跳过**：确认点 2 是流程硬性节点，未获用户确认不得进入配图环节。
9. **回退保留上游**：用户要求重做时只重做初稿，大纲与调研保留不动。
