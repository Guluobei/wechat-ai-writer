# 模块 4：正文撰写（Writer）

> 调性适配 + 风格档案 + 实时事实核查。按大纲和素材生成初稿。★ 确认点 2

## 模块职责

依据大纲与调研素材，参照风格档案生成完整初稿。写作过程中实时事实核查，处理专业术语，控制过渡与节奏。输出后暂停，等待用户确认（确认点 2）。

## 输入

| 输入项 | 说明 |
|--------|------|
| 大纲 | `ArticleProject.outline`，含结构、节奏、配图标注、钩子与收束 |
| 调研报告 | `ArticleProject.research`，含事实、数据、观点、案例、引用 |
| 风格档案 | `assets/default-style-profile.md`，AI 行业通用调性 |

> 前置校验：若无大纲，提示「撰写需要大纲。先运行大纲构建，或直接告诉我文章结构」。

## 输出

完整初稿，写入 `ArticleProject.draft` 字段，结构如下：

```json
{
  "content_markdown": "完整初稿 Markdown 文本",
  "word_count": 3500,
  "inline_citations": [
    {"ref": "来源名称", "url": "链接", "position": "段落位置描述"}
  ]
}
```

- `content_markdown`：完整初稿正文，Markdown 格式，含内联引用标记（如 [1]、[2]）
- `word_count`：正文字数统计
- `inline_citations`：内联引用列表，每条含来源名称、链接和位置描述，供模块6排版时生成引用标注

## 执行步骤

### 1. 调性适配

根据文章类型切换写作风格：

| 文章类型 | 调性倾向 |
|----------|----------|
| `tech_analysis` | 专业、准确、有深度，兼顾可读 |
| `product_review` | 客观、体验导向、有观点但不偏颇 |
| `science_explainer` | 通俗、生动、善用类比，降低理解门槛 |
| `industry_insight` | 宏观、有判断力、逻辑严密 |

### 2. 风格档案参照

始终参照 `assets/default-style-profile.md`，保证跨文章调性一致。若存在用户级风格档案（`/workspace/config/wechat-ai-writer/style-profile.md`），优先采用并覆盖默认项。

### 3. 素材织入

- 将事实、数据、案例自然织入正文，避免堆砌罗列。
- 引用以公众号友好方式呈现：行文内自然提及来源，关键数据可在句末简注出处，不使用学术式脚注。
- 观点性内容须标注来源或持有者，不把他人观点写成自家结论。

### 4. 专业术语处理

- AI 术语首次出现时加注简短解释（括号或同位语）。
- 重复出现不再重复加注。
- 面向泛读者的科普类文章，术语解释应更通俗。

### 5. 实时事实核查

- 写作过程中实时核对引用与调研报告是否一致。
- 引用无出处或与调研报告冲突的，标记「待核实」。
- 标记位置清晰可辨，便于后续校对模块（模块 7）复核。

### 6. 过渡与节奏

- 控制段落长度，避免大段文字墙。
- 章节间过渡自然，符合大纲节奏设计（篇幅占比）。
- 信息密度适中，重点处适当展开，铺垫处适度精简。

## 降级方案

- **事实核查无引用支撑**：标记「待核实」继续写作，不中断流程，并在初稿交付时告知用户「第 N 段数据未找到出处，已标注待核实」。

## 门控：确认点 2

初稿输出后**必须暂停**，向用户展示并等待确认：

```
📝 初稿已完成（约 XXXX 字），请确认：
  [展示初稿全文或摘要]
  → 回复「确认」继续配图
  → 回复「修改：XX」调整初稿
  → 回复「重做」重新撰写
```

- 用户「确认」：进入模块 5 配图。
- 用户「修改：XX」：按反馈调整指定部分后重新展示。
- 用户「重做」：保留大纲与调研，重新撰写。

## 注意事项

1. **风格档案优先级**：用户级风格档案 > 默认风格档案，二者冲突时以用户级为准并保持一致。
2. **不脱离大纲**：正文须覆盖大纲所有章节，新增内容不得偏离大纲结构；如确需调整结构，应提示用户回到模块 3。
3. **不编造素材**：调研报告未提供的事实、数据、案例不得自行杜撰，缺素材处标记「待核实」或留出占位。
4. **引用必须可溯**：正文中出现的所有数据与关键事实，须能在调研报告引用列表中找到出处。
5. **术语加注仅首次**：避免重复加注造成阅读打断。
6. **「待核实」要显眼**：标记格式统一且醒目，便于校对环节集中处理。
7. **门控不可跳过**：确认点 2 是流程硬性节点，未获用户确认不得进入配图环节。
8. **回退保留上游**：用户要求重做时只重做初稿，大纲与调研保留不动。
