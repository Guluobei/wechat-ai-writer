# 模块 2：调研收集（Researcher）

> 双通道采集 + 竞品扫描 + 引用管理。围绕选题收集素材，整理成带引用的结构化调研报告。

## 模块职责

围绕选题方案收集素材，对信源分级、去重、冲突检测，并扫描竞品内容识别已覆盖角度与空白点，最终整理成带引用出处的结构化调研报告，供大纲与撰写模块使用。

## 输入

| 输入项 | 说明 |
|--------|------|
| 选题方案 | `ArticleProject.topic`，含暂定标题、核心论点、目标读者、重点调研方向 |

> 前置校验：若无选题，提示「调研需要先有选题。告诉我你想写什么主题，或输入『自动选题』」。

## 输出

调研报告，写入 `ArticleProject.research` 字段，包含以下子字段：

- `sources`：使用到的信源清单
- `facts`：关键事实
- `data_points`：可用数据
- `opinions`：核心观点
- `cases`：典型案例
- `competitor_coverage`：竞品覆盖分析
- `citations`：引用列表（来源名 + 链接 + 检索时间）

报告正文结构：选题背景、关键事实、核心观点、可用数据、典型案例、竞品覆盖分析、引用列表。

## 执行步骤

### 1. 双通道采集

- **固定信源通道**：按选题关键词匹配 `assets/default-sources.json` 中的信源，使用 `WebFetch` 工具定向抓取相关内容。
- **临时 Web 搜索通道**：围绕选题使用 `WebSearch` 工具展开 Web 搜索，补充固定信源未覆盖的信息。
- 两路素材汇入统一素材池，标注来源通道。

### 2. 信源权威性分级

对每条素材的信源按四级分级，分级影响后续取舍权重：

| 级别 | 说明 | 示例 |
|------|------|------|
| 一手官方 | 官方公告、论文、技术文档、产品发布页 | 官方博客、arXiv |
| 权威媒体 | 行业权威媒体的报道与解读 | 科技媒体、行业垂媒 |
| 社区讨论 | 社区、论坛、技术社区的讨论 | 开发者社区、问答站 |
| 个人观点 | 个人博客、自媒体评论 | 个人号、专栏 |

### 3. 竞品内容扫描

- 搜索同主题已发布的公众号 / 自媒体文章。
- 识别竞品已覆盖的角度、论点、数据。
- 标记**内容空白点**：竞品未触及或浅尝辄止的角度，作为本文差异化切入点。

### 4. 信息去重与冲突检测

- **去重**：多源报道同一事实时合并，保留最权威来源。
- **冲突检测**：多源对同一事实表述矛盾时，标记「待核实」，并列出冲突版本与各自出处，不擅自裁决。

### 5. 结构化整理

将素材按四类归类，每条附引用出处（来源名 + 链接 + 检索时间）：

| 类别 | 内容 |
|------|------|
| 事实 | 可验证的客观陈述、事件、动态 |
| 观点 | 各方解读、判断、预测（须标注观点持有者） |
| 数据 | 数字、指标、对比、统计（须标注出处与口径） |
| 案例 | 具体产品、应用、事件实例 |

### 6. 输出调研报告

按以下结构输出调研报告：

1. **选题背景**：热点脉络、为何此时值得写
2. **关键事实**：已核实的事实清单
3. **核心观点**：各方观点汇总，含分歧点
4. **可用数据**：数据点清单，含口径与出处
5. **典型案例**：可用于佐证或展开的案例
6. **竞品覆盖分析**：已覆盖角度 + 内容空白点
7. **引用列表**：完整出处（来源名 + 链接 + 检索时间）

## 降级方案

- **Web 搜索失败**：只用固定信源完成调研，告知用户「Web 搜索暂不可用，已用固定信源完成调研」。
- **某信源抓取失败**：跳过该信源，告知用户「XX 信源暂不可访问，已跳过，其余正常」。
- **素材严重不足**：如实告知，建议调整选题或放宽调研范围，不强行拼凑。

## 注意事项

1. **每条素材必须可溯源**：事实、数据、观点都要附出处，无出处的素材标注「待核实」，不混入正式事实。
2. **冲突不裁决**：事实矛盾只标记待核实并列出双方出处，由用户或撰写环节判断。
3. **观点与事实分离**：报告中专人观点不得伪装成客观事实。
4. **数据标注口径**：数据点须注明统计口径、时间范围、来源，避免误用。
5. **竞品扫描要诚实**：竞品覆盖分析是差异化的基础，不能为凸显本文而贬低或无视已有优质内容。
6. **检索时间必须记录**：信源内容会更新，引用须附检索时间以便复核。
7. **输出结构完整**：`research` 字段子项（sources/facts/data_points/opinions/cases/competitor_coverage/citations）需齐全，缺项以空值占位并说明原因。
