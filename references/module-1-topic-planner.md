# 模块 1：选题策划（Topic Planner）

> 热点追踪 + 角度建议。从 AI 行业热点中找到值得写的选题。

## 模块职责

从 AI 行业热点中识别值得写的选题，为每个选题提供切入角度与调研方向，输出可供后续模块（调研、大纲、撰写）直接使用的选题方案。

聚焦领域：产品、技术、科普、行业解读四类 AI 内容。

## 输入

| 输入项 | 说明 | 是否必填 |
|--------|------|---------|
| 领域关键词 | 用户指定的 AI 细分方向，如「多模态」「Agent」「具身智能」 | 可选 |
| 空输入 | 用户未指定关键词时，自动扫描全局 AI 热点 | 自动 |

## 输出

选题方案文档，写入 `ArticleProject.topic` 字段。

每个选题方案包含：暂定标题、核心论点、目标读者、预计篇幅范围、重点调研方向。

选题方案写入 `ArticleProject.topic` 字段，含 `title`、`content_type`、`core_argument`、`target_audience`、`estimated_length`、`research_directions` 六个子字段。

## 执行步骤

### 1. 扫描热点

- 加载 `assets/default-sources.json` 中的固定信源，按关键词（用户给定或自动推断）匹配相关信源进行扫描。
- 同时使用 `WebSearch` 工具发起 Web 搜索，补充分固定信源未覆盖的近期 AI 热点。
- 识别近期（建议覆盖近 7–14 天）出现的 AI 热点，形成候选选题池。

### 2. 选题打分与筛选

对每个候选选题按以下四个维度打分，综合排序后筛出 Top 3–5：

| 维度 | 说明 |
|------|------|
| 时效性 | 热点发生时间、是否处于发酵期或衰减期 |
| 话题热度 | 讨论量、传播范围、跨平台热度 |
| 受众价值 | 对目标读者的信息增量与实用价值 |
| 差异化空间 | 现有内容是否已充分覆盖、本文能否做出增量 |

### 3. 切入角度建议

为每个选题提供切入角度建议，从以下类型中选择最契合的 1–2 种：

- **技术解读**：拆解原理、架构、关键创新点
- **产品分析**：功能、体验、定位、商业模式
- **科普**：面向非专业读者解释概念与影响
- **行业影响**：对产业格局、竞争态势、趋势走向的判断

### 4. 输出选题方案

为每个入选选题输出完整方案：

- **暂定标题**：1–2 个候选，符合公众号传播特性
- **核心论点**：一句话说清本文想传达什么
- **文章类型**：从 `tech_analysis`（技术解读）/ `product_review`（产品分析）/ `science_explainer`（科普）/ `industry_insight`（行业解读）中选择最契合的类型
- **目标读者**：明确读者画像（从业者/关注者/泛读者）
- **预计篇幅范围**：如 2000–3000 字
- **重点调研方向**：列出后续调研需要重点查证的事实与数据

## 降级方案

- **Web 搜索失败**：只用固定信源扫描热点，并向用户告知「网络搜索暂不可用，已用内置信源选了 N 个热点」。
- **固定信源加载失败**：仅依赖 Web 搜索，并告知用户信源加载异常。
- **候选选题不足 3 个**：如实告知热点偏少，放宽时间窗口或扩大关键词范围重新扫描。

## 注意事项

1. **不凭空编造热点**：所有候选选题必须有信源或搜索结果支撑，无法溯源的选题剔除。
2. **时效优先但不追过期热点**：已明显衰减的热点降权，除非有新进展或新角度。
3. **差异化是核心**：打分时「差异化空间」权重应高于纯热度，避免扎堆已写烂的题材。
4. **标题仅为暂定**：最终标题在发布模块（模块 8）确定，此处不强求完美。
5. **尊重用户意向**：用户给定关键词时，扫描结果应围绕该关键词，不要跑题到无关热点。
6. **输出可被直接消费**：选题方案的字段需与后续模块期望一致，避免结构缺失导致流程中断。
