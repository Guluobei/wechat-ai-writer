# 模块 8：发布分发（Publisher）★ 确认点 3

> 生成发布所需的全部元信息：封面缩略图、标签、摘要、发布时机、候选标题、发布前清单。输出后暂停，等待用户确认（确认点 3）。

## 模块职责

- 生成发布所需的全部元信息，打包成「发布就绪包」
- 基于历史经验与通用规律给出发布时机建议
- 生成 3-5 个候选标题（直述型 / 悬念型 / 数字型）
- 发布前逐项确认清单
- 输出后暂停，等待用户确认（确认点 3）

## 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 成品 HTML | `ArticleProject.formatted` | 用于摘要撰写参考、发布前清单确认 |
| 配图清单 | `ArticleProject.images` | 封面图 cover 用于生成缩略图版本 |
| 文章主题与类型 | `ArticleProject.topic` | 主题、内容类型（content_type），用于标签与标题生成 |
| 复盘历史经验（可选） | `/workspace/config/wechat-ai-writer/history.json` | 历史发布时机、标题风格表现数据，用于时机与标题建议 |
| 用户级历史（可选） | 项目级配置覆盖 | 项目级历史经验（如有） |

历史经验缺失时用通用规律，不阻塞流程。

## 输出

发布就绪包写入 `ArticleProject.publish` 字段，结构如下：

```json
{
  "cover_thumbnail": {
    "path": "/workspace/projects/<article_id>/images/cover-thumb.png",
    "status": "success | fallback"
  },
  "tags": ["AI", "大模型", "..."],
  "summary": "120 字内摘要",
  "publish_time_suggestion": {
    "recommended": "工作日 20:00",
    "reason": "建议依据（历史数据 / 通用规律）",
    "alternatives": ["工作日 08:00", "周末 10:00"]
  },
  "title_candidates": [
    { "style": "直述型", "title": "..." },
    { "style": "悬念型", "title": "..." },
    { "style": "数字型", "title": "..." }
  ],
  "pre_publish_checklist": [
    { "item": "成品 HTML", "status": "ready" },
    { "item": "封面图", "status": "ready | missing" },
    { "item": "摘要", "status": "ready" },
    { "item": "标签", "status": "ready" },
    { "item": "标题", "status": "ready" },
    { "item": "发布时间", "status": "ready" }
  ],
  "confirmed": false
}
```

`confirmed: false` 表示尚未经用户确认；用户确认后置为 `true`，流程进入模块 9 复盘准备。

## 执行步骤

### 步骤 1：封面元信息

- 从配图封面图 `images.cover` 生成缩略图版本（公众号封面缩略图规格）
- 确认最终封面：若封面图为 `placeholder` / `skipped` 状态，标注 missing 并在清单中提示
- 缩略图保存到 `images/cover-thumb.png`

### 步骤 2：标签生成

- 生成 3-5 个公众号标签
- 覆盖 AI 细分领域关键词（如大模型、多模态、Agent、具身智能、AI 应用等）
- 兼顾主题词与流量词：既准确反映内容，也覆盖读者搜索习惯
- 标签来自文章主题与类型，不杜撰与内容无关的蹭热度标签

### 步骤 3：摘要撰写

- 120 字以内
- 准确概括文章核心，同时吸引点击
- 不做标题党：摘要承诺与正文一致，不夸大、不误导
- 避免与候选标题完全重复

### 步骤 4：发布时机建议

综合历史数据与通用时段规律给出推荐：

- **有历史数据**：从 `history.json` 分析历史最佳发布时段（按内容类型、完读率、转发表现），给数据支撑的推荐
- **无历史数据**：用通用时段建议（工作日早 8 点或晚 8 点）
- 给出推荐时段 + 备选时段，并写明建议依据

### 步骤 5：标题优化

生成 3-5 个候选标题，覆盖不同风格：

| 风格 | 特点 |
|------|------|
| 直述型 | 准确概括主题，信息明确 |
| 悬念型 | 制造好奇，吸引点击但不标题党 |
| 数字型 | 用数字/盘点结构增强可读性 |

- 标题需与正文一致，不夸大、不歪曲
- 兼顾搜索友好与点击吸引
- 标注每个候选的风格类型，便于用户选择

### 步骤 6：发布前清单

逐项确认发布就绪状态：

- 成品 HTML（`final.html`）
- 封面图（含缩略图）
- 摘要
- 标签
- 标题（候选）
- 发布时间（建议）
- [ ] 图片上传：共 N 张图片需手动上传到公众号编辑器（清单见排版模块输出）

任一项 missing / 未就绪在清单中标注，提示用户补齐。清单汇总后进入门控。

## 降级方案

| 失败点 | 自动降级 | 用户感知（三段式） |
|--------|---------|------------------|
| 无历史数据 | 用通用时段建议 | 「暂无历史数据，建议工作日早 8 点或晚 8 点发布」 |
| 封面图缺失（placeholder/skipped） | 清单标注 missing，提示补图 | 「封面图未就绪，请发布前补图，其余已备齐」 |
| 历史经验文件读取失败 | 用通用规律，不阻塞 | 「历史经验读取失败，已用通用时段建议」 |

## 门控：确认点 3

本模块输出后**必须暂停**，向用户展示发布就绪包并等待确认：

```
📦 发布就绪包已准备好，请确认：
  [展示成品 HTML 预览链接、封面图、标题候选、摘要、标签、发布时机]
  → 回复「确认」完成流程
  → 回复「修改：XX」调整指定元素
```

- 用户回复「确认」→ `confirmed: true`，流程进入模块 9 复盘准备
- 用户回复「修改：XX」→ 按要求调整指定元素后重新展示
- 不确认不进入下一步
- **直接输出成品 HTML**：确认点 3 展示时，直接在对话中输出完整成品 HTML 内容（或提供可一键复制的文件链接），方便用户直接复制粘贴到公众号编辑器

## 注意事项

1. **门控必须暂停**：确认点 3 是全流程最后一个关键节点，输出后必须暂停等用户确认，不可自动推进
2. **标题不标题党**：候选标题可吸引点击但承诺须与正文一致，不夸大、不歪曲、不误导
3. **标签不蹭无关热度**：标签需与内容相关，不为流量堆砌无关热词
4. **摘要与正文一致**：摘要承诺须在正文兑现，120 字内准确概括
5. **时机建议给依据**：推荐时段要写明依据（历史数据 / 通用规律），让用户可判断
6. **清单逐项可执行**：发布前清单每项都要有明确状态，missing 项要提示用户怎么补
7. **历史经验只读不改**：本模块只读取 `history.json` 用于建议，写入由模块 9 复盘负责
8. **降级不阻塞**：封面缺失、无历史数据等都走降级，标注清楚即可，不卡住确认流程
9. **粘贴后检查清单**：用户将 HTML 粘贴到公众号编辑器后，建议检查以下项：标题样式是否生效、代码块是否正常显示、图片是否完整加载、引用标注是否清晰、移动端预览是否正常
10. **图片需手动上传**：公众号编辑器不支持本地图片路径，用户粘贴 HTML 后需逐张上传图片。排版模块已生成图片上传清单，发布前确认所有图片已上传
